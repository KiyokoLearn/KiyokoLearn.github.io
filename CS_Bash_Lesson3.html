Lesson 3 -- Files

Note:
In the previous tutorial, when showing what commands to type into the terminal, I wrote out Username@ComputerName~$ but I get tired fo typing that whole thing out over and over again. From now on when I am showing you commands to type in, I will simply use $. I am doing this because the prompt changes depending on what directory you are in, etc. Just know that when I use $, I am implying that there is a prompt before the $.


Echo

The previous tutorial was about just navigating through files, but what if I want to interact with the shell more? Well, the first command we'll talk about is echo. The echo command basically tells the shell to "echo" back whatever you tell it to.

For example, if I want the computer to say to me "Hello World":
$ echo "Hello World"



Then press enter and it will spit out:
Hello World



More Than One Command

What if we want the shell to do multiple things at once, instead of us having to type in each individual command and hit enter? You can use the ";" to separate commands. For example, I want to the computer to spit out "Hello World" and "Hello Kiyoko":
$ echo "Hello World";echo "Hello Kiyoko"



When you press enter, the terminal prints out:
Hello World
HelloKiyoko



New Line with Echo

But what if I'm the perfectionist who doesn't want "Hello World" and "Hello Kiyoko" on two different lines? What if I want them on the same line? To do this, we would use -n with echo. The -n is telling Bash to not use a new line after that command

For Example:
$ echo -n "Hello World";echo "Hello Kiyoko:



When you press enter:
Hello WorldHello Kiyoko



Saving to a File

So, you can now print to the screen. That's cool. But let's say my laptop just died. When I open the terminal again, everything I printed to the screen is gone! Does that mean everything I printed to the screen earlier is gone? Do I have to retype EVERYTHING?! Well, there are two fixes for this. The first is that by using the up arrow, you can recall previously entered commands. The second is that, when using the echo command, instead of having it print to the screen, you can have it save to a file. To do this, you will use >> but we will come back to this. However, I first want to mention that you can create a file using the touch command.
$ touch filename1



now, if you were to type in ls, the file filename1 would be listed, even though the file is empty But I don't want that file to be empty. As mentioned before, I'm going to use the echo command with >>
$ echo "Hello World" >> filename1



This is telling Bash to take the input and instead of outputting the result to the screen, output the info to a file.

But what if I forgot to use the touch command first! what if I typed in:
$ echo "Hello World" >> filename2



Don't freak out. If filename2 didn't already exist when you executed that command, It simply creates the file filename2.



Removing Files

But, but, but.... what if I created a file and didn't mean to? How do I get rid of it? You can remove files using the rm command.
$ rm filename2



Keep in mind that, unlike other systems, Bash does not prompt you to make sure you really want to delete that file. Once you delete that file, its gone for good. Also know that rm is only for deleting files. It does not work for directories.
